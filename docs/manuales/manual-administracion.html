Gestión de Departamentales

Acceso: Gestión → Departamentales

  • Crear: nombre, código (abreviatura), activo
  • Editar / Desactivar: sin eliminar registros asociados
Ejemplo

Para crear la departamental de Alta Verapaz, ingrese Nombre = "Alta Verapaz", Código = "AV". Quedará activa por defecto para que los usuarios puedan asociarle actividades.

Gestión de Usuarios

Acceso: Accesos → Usuarios

Crear usuario

  1. Haz clic en Crear
  2. Completa: foto (opcional), username, email, nombres, apellidos, contraseña, oficina departamental, rol

Roles disponibles

RolPermisos

Imagen: Listado de usuarios del sistema.

Suplantación (Impersonate)

  1. En la lista de usuarios, haz clic en Impersonate
  2. Confirmas la acción
  3. El sistema te muestra el panel como si fueras ese usuario
  4. Para volver: menú superior derecho → Salir de la departamental

Roles y Permisos (Filament Shield)

Acceso: Accesos → Roles

Cada rol tiene permisos granulares sobre recursos, páginas y widgets.

Permisos por recurso

  • view — Ver registros
  • view_any — Ver todos (incluso de otras departamentales)
  • create — Crear nuevos
  • update — Editar existentes
  • delete / delete_any — Eliminar

Cierre Mensual

Consolida las actividades de una departamental en un mes y genera un PDF.

Generar cierre individual

Desde Actividades: filtra por departamental → Generar Cierre Mensual → selecciona mes y año.

Desde Cierres Mensuales: Generar Cierre Mensual → selecciona departamental, mes y año.

El sistema:

  1. Cuenta actividades proyectadas, ejecutadas, pendientes y canceladas
  2. Calcula el % de cumplimiento
  3. Genera un PDF con el resumen por programa
  4. Asocia las actividades al cierre
Una vez generado el cierre, las actividades de ese mes NO se pueden editar ni eliminar.
Ejemplo

Si la departamental de Sacatepéquez tuvo 15 actividades en junio (12 ejecutadas, 3 pendientes), al generar el cierre mensual el sistema calculará un cumplimiento del 80%, generará un PDF con el desglose por programa y bloqueará las 15 actividades contra ediciones.

Generar consolidado

En Actividades → Descargar Consolidado → selecciona mes y año → PDF con todas las departamentales.

Seguridad

  • Autenticación: contraseñas hasheadas con bcrypt
  • Roles y permisos: control granular por recurso (Shield)
  • Suplantación: solo super_admin puede impersonar
  • Notificaciones: canal seguro por base de datos
  • Logs de actividad: seguimiento de cambios (spatie/activitylog)

Preguntas Frecuentes

1. ¿Cómo creo un usuario nuevo?
Vaya a Accesos → Usuarios → Crear. Complete los campos obligatorios (username, email, nombres, apellidos, contraseña) y asigne un rol. El usuario recibirá sus credenciales de forma segura.
2. ¿Qué hago si un usuario olvida su contraseña?
Puede restablecerla desde Accesos → Usuarios, edite el usuario y escriba una nueva contraseña. El sistema la guardará de forma segura.
3. ¿Puedo eliminar un usuario?
No se recomienda eliminar usuarios porque pierde la trazabilidad. En su lugar, desactive la cuenta desde la edición del usuario (marque como inactivo).
4. ¿Qué significa “Reabrir” un cierre mensual?
Permite desbloquear las actividades de un mes ya cerrado para hacer correcciones. Solo debe usarse cuando sea estrictamente necesario, ya que afecta los reportes históricos.
5. ¿Cómo configuro el correo SMTP?
Vaya a Configuración → Correo. Ingrese los datos del servidor SMTP (host, puerto, usuario, contraseña). Use la opción "Probar Envío" para verificar la conexión antes de guardar.
6. ¿El sistema tiene respaldo automático de datos?
El sistema no incluye respaldos automáticos integrados. Es responsabilidad del administrador del servidor configurar copias de seguridad periódicas de la base de datos.
